## 一、产品概述

### 1.1 产品定位

**一句话定位**：AI 驱动的全方位英语学习助手，让英语学习更高效、更有趣

**产品愿景**：通过 AI 技术帮助用户提升英语能力，包括词汇积累、阅读理解、口语练习等多个维度，让学习过程个性化、高效化。

### 1.2 问题陈述

| 维度 | 描述 |
|------|------|
| **目标用户** | 英语学习者，希望提升词汇量、阅读能力和口语表达能力 |
| **核心痛点** | 生词记不住（缺乏语境）；阅读材料单一（没有个性化）；口语没人练（缺乏语伴和环境）；学习效率低（工具分散） |
| **期望结果** | 一站式英语学习平台，词汇-阅读-口语闭环，AI 个性化辅导 |

## 二、功能详情

| D002 | 音标显示 | 显示美式/英式音标，支持播放发音 | ✅ 已完成 |
| D003 | 中英文释义 | 同时展示中文释义和英文释义 | ✅ 已完成 |
| D004 | 例句展示 | 展示单词在句子中的用法 | ✅ 已完成 |
| D005 | 近义词 | 展示近义词及区别说明 | ✅ 已完成 |
| D006 | 反义词 | 展示反义词 | ✅ 已完成 |
| D007 | 自动保存 | 查询的单词自动保存到生词本 | ✅ 已完成 |
| D008 | 词典数据 | 使用免费词典 API 获取音标、释义等 | ✅ 已完成 |

#### 技术实现

- **词典 API**: 使用 Free Dictionary API 获取基础词典数据
- **LLM 增强**: 使用智谱 GLM 模型生成中文释义、例句、近义词
- **流式输出**: 采用流式生成，逐步展示结果

### 3.4 交互流程

#### 模式一：按住说话

```
┌─────────────────────────────────────────────────────┐
│                    对话界面                          │
├─────────────────────────────────────────────────────┤
│                                                      │
│    [AI 消息气泡 - 文字 + 播放状态]                   │
│                                                      │
│    [用户消息气泡 - 文字]                             │
│                                                      │
│    [AI 消息气泡 - 正在输入...]                       │
│                                                      │
├─────────────────────────────────────────────────────┤
│                                                      │
│           ╭──────────────────────╮                  │
│           │   按住说话           │                  │
│           │   (显示时长)         │                  │
│           ╰──────────────────────╯                  │
│                                                      │
│           [切换到自由对话模式]                       │
│                                                      │
└─────────────────────────────────────────────────────┘

交互步骤：
1. 用户按住按钮 → 开始录音，显示时长
2. 用户说话 → 实时语音识别
3. 用户松开 → 发送语音，显示"识别中..."
4. ASR 识别完成 → 显示用户文字消息
5. AI 开始回复 → 显示 AI 消息气泡 + 播放语音
6. 语音播放完成 → 等待用户下一次输入
```

```
│    [AI 消息气泡]                                     │
│                                                      │
│    [用户消息气泡]                                    │
│                                                      │
│    [AI 正在说话 - 动画指示]                          │
│                                                      │
├─────────────────────────────────────────────────────┤

交互步骤：
1. 用户点击"自由对话" → 进入模式
2. 系统持续监听麦克风 → VAD 检测用户是否在说话
3. 检测到用户说话 → 录音，识别
4. 用户说完（静音 2 秒）→ 发送识别结果
5. AI 开始回复 → 播放语音
6. AI 播放完成 → 继续监听
7. 用户点击"结束对话" → 退出模式，生成反馈
```

### 4.5 Safari 音频自动播放解决方案

Safari 要求音频在用户交互的同步上下文中播放。

**解决方案：**
1. 使用 Web Audio API 而非 HTML Audio 元素
2. 在用户点击/触摸事件中调用 `unlockAudio()` 解锁 AudioContext
3. 解锁后即可异步播放音频

**关键代码：**
```typescript
// 解锁音频播放（必须在用户交互的同步上下文中调用）
function unlockAudio(): void {
  const ctx = getAudioContext();
  if (ctx.state === 'suspended') {
    ctx.resume();
  }
  // 创建静音振荡器确保音频上下文被激活
  const oscillator = ctx.createOscillator();
  const gainNode = ctx.createGain();
  gainNode.gain.value = 0.001; // 几乎静音
  oscillator.connect(gainNode);
  gainNode.connect(ctx.destination);
  oscillator.start();
  oscillator.stop(ctx.currentTime + 0.001);
}
```
